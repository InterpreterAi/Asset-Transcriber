The project has been reverted to the previous stable version because the last update modified segmentation behavior and broke speaker diarization.

We now want to keep the stable segmentation logic exactly as it was and only apply the following fixes.

IMPORTANT:
Do NOT change segmentation logic, transcription streaming, Soniox token handling, or speaker diarization behavior.

Segments must continue to finalize only when:
• a speaker change occurs
• the silence timer is reached

Do not introduce adaptive speech-speed logic again.

------------------------------------------------

1. Reduce the silence timer slightly

Change the silence threshold used for segment finalization from 1.2 seconds to 0.9 seconds.

This is only to make segments finalize slightly faster.

Do not modify any other segmentation behavior.

------------------------------------------------

2. Prevent transcript loss when STOP is pressed

Current issue:
If the user presses STOP while the transcript is still grey (non-final tokens), the text disappears.

Required behavior:
When STOP is pressed:
• immediately finalize the current grey segment
• convert it into a finalized segment
• trigger the final translation
• keep the transcript visible

The transcript must never disappear when stopping.

------------------------------------------------

3. Fix translation language selection

Translation must always respect the user-selected language.

Correct logic:

sourceLang = detected language  
targetLang = userSelectedLanguage

Remove any hardcoded English or Arabic behavior.

------------------------------------------------

4. Enable translation for all supported languages

All languages must behave exactly the same way English ↔ Arabic currently works.

Examples:
English → French  
French → Arabic  
Spanish → English  
German → Turkish  

Changing the language selector must affect the next segment immediately.

------------------------------------------------

Again, do NOT modify:
• segmentation logic
• diarization behavior
• transcription streaming
• Soniox token processing

Only apply the fixes listed above.
