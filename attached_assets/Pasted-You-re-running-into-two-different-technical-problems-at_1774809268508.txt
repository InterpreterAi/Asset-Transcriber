Problem 1 — You are rendering partial transcripts as final rows

Your system shows this:

Speaker 1
Look like, you know, relatively normal.

Speaker 1
Look like, you know, relatively normal, but I didn't talk to anybody.

That means every streaming update is printed as a new message.

Soniox does not do that.

Soniox behavior:

Speaker 1
Look like, you know, relatively normal, but I didn't talk to anybody.

It updates the same row until the phrase ends.

Your UI must treat streaming transcripts like this:

PARTIAL → update row
PARTIAL → update row
PARTIAL → update row
FINAL → lock row
NEW SENTENCE → create new row

Right now your code does:

PARTIAL → create row
PARTIAL → create row
PARTIAL → create row

That’s why everything looks chaotic.

⸻

Problem 2 — You are displaying speaker diarization too early

This is the bigger issue you’re seeing.

Speaker diarization does NOT stabilize immediately.

The AI needs a few seconds of voice context to understand the speakers.

So early results look like:

Speaker 1
Speaker 1
Speaker 2
Speaker 1
Speaker 2

Then after a moment the model corrects itself.

That’s why your screen “fixes itself later”.

Soniox solves this by delaying speaker labeling slightly.

They buffer a few seconds before finalizing the speaker.

⸻

What Soniox actually does internally

Real pipeline:

audio
 ↓
live partial text
 ↓
temporary speaker guess
 ↓
buffer 2–3 seconds
 ↓
speaker stabilized
 ↓
final transcript row

Your app skips the buffer.

So the UI prints unstable speaker guesses.

⸻

The fix (this is the important part)

You need three rules in your transcript engine.

⸻

Rule 1 — Only one live row

There must be one active row.

Example:

let activeSegment = null;


⸻

Rule 2 — Update partial results

if result.is_final == false
update activeSegment text

Do NOT create new rows.

⸻

Rule 3 — Wait before locking speaker

Buffer about 1.5–2 seconds.

Example logic:

if (result.is_final) {

  setTimeout(() => {
     finalizeSegment(activeSegment)
     activeSegment = null
  }, 1500)

}

This allows diarization to stabilize.

Why Soniox feels instant even with buffering

Because they still show text immediately, but the speaker label locks later.

You see text instantly but speaker becomes correct.
