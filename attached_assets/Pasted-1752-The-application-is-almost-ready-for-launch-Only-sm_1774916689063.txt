BUG 1 — TRANSCRIPT DISAPPEARS AFTER STOP

When the user presses STOP, any grey (non-final) transcript segments are removed while the translation remains visible.

This causes the transcript and translation to become inconsistent.

Fix:

When STOP is pressed:

• finalize any grey segments
• keep both transcript AND translation visible
• do NOT clear transcript text
• do NOT delete segments

Pseudo behavior:

if (stopPressed) {
    finalizeOpenSegments()
    preserveTranscriptHistory()
}

Transcript and translation history must remain visible after stopping.

------------------------------------------------

BUG 2 — NEW SEGMENT AFTER PAUSE

If the user pauses for more than ~1 second and speaks again, a new segment should begin.

Currently the system continues writing into the previous segment.

Fix:

Add pause-based segmentation.

Example logic:

if (speechPause > 900ms) {
    closeCurrentSegment()
    startNewSegment()
}

This should not affect diarization or streaming.
